Algorithm Fundamentals

An algorithm is a step-by-step procedure to solve a problem. For an algorithm to be effective, it must possess several key characteristics:

  • Input: Takes zero or more well-defined inputs.
  • Output: Produces one or more well-defined outputs.
  • Definiteness: Each step is clear and unambiguous.
  • Finiteness: Terminates after a finite number of steps.
  • Effectiveness: Each step is basic and executable.

Arrays: The Foundation

An array is a collection of items of the same type stored in contiguous memory locations. It's the bedrock of many other data structures.

Advantage: Fast Access (O(1))

Accessing any element by its index is instantaneous because its memory address can be calculated directly.

Disadvantage: Fixed Size

The size of an array is fixed. Resizing requires creating a new array and copying all elements, which is an inefficient O(n) operation.

Array Memory Layout: Row vs. Column Major

How a 2D array (matrix) is stored in linear memory. This affects performance, especially for large datasets.

Searching Algorithms: Linear vs. Binary

Finding data is a common task. Linear Search is simple but slow, checking every item. Binary Search is much faster but has a critical requirement: the data must be sorted beforehand.

Requirement Works on ANY data (sorted or unsorted). Data MUST be sorted.
